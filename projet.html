Poki

Poki est un mini-jeu de combat inspiré des jeux classiques de combat, il revisite l’univers Pokémon. Le joueur sélectionne deux Pokémon et les fait s’affronter dans une arène animée, avec une interface rétro et des effets visuels dynamiques. Le projet met l’accent sur le design avec des barres de vie animées et une mise en scène du combat rappelant les jeux de versus. Les animations et transitions renforcent l’immersion et donnent un style rétro-gaming moderne.


Technologies utilisées


  • HTML5

    • structure de l’interface (arène, HUD, sélection des Pokémon, boutons d’actions).


  • CSS3

    • Style rétro, animations, transitions, barres de vie et effets visuels.


  • JavaScript (ES6+)

    • logique du combat, gestion des états du jeu, manipulation dynamique du DOM et interactions utilisateur.


  • API Pokémon (PokéAPI)

    • récupération dynamique des données des Pokémon (statistiques, types, sprites) via requêtes fetch.


  • JSON

    • exploitation et traitement des données reçues pour alimenter la logique du jeu.


  • Asynchrone

    • utilisation de async / await pour un chargement fluide sans rechargement de page.

MaTodolist

Une application de type “todolist” progressive (PWA) permettant de créer, marquer comme terminées et supprimer des tâches, avec la possibilité de recevoir des notifications de rappel. L’application fonctionne aussi bien en ligne dans un navigateur qu’installée sur mobile, offrant une expérience proche d’une application native.


Technologies utilisées :


  • HTML :

    • structure de la page et éléments interactifs (input, boutons, liste).


  • JavaScript :

    • gestion dynamique des tâches, stockage local (localStorage), interactions utilisateurs, notifications et logique de l’application.


  • Service Worker:

    • mise en cache des fichiers pour permettre l’utilisation hors ligne et gestion des mises à jour de la PWA.


  • Manifest.json :

    • configuration de la PWA (icônes, thème, start_url, affichage standalone).

Anonym_Avis&Comment

Projet web interne pour une école permettant aux étudiants de partager des avis anonymes sur les cours et enseignants. L’objectif est d’améliorer * la qualité pédagogique grâce à des retours constructifs, tout en garantissant l’anonymat total des étudiants. Le site propose une interface simple et intuitive pour les étudiants et un tableau de bord sécurisé pour l’administration.


Technologies utilisées


  • HTML / CSS :

    • Structure et style des pages web, interface responsive et ergonomique.


  • JavaScript :

    • Interactivité côté client, gestion des listes dynamiques et validation de formulaire.


  • PHP :

    • Traitement des formulaires, communication sécurisée avec la base de données.


  • SQL (PostgreSQL via Supabase) :

    • Stockage des cours, étudiants, commentaires et administrateurs, gestion sécurisée des données.


  • PDO (PHP) :

    • Connexion sécurisée à la base de données pour éviter les injections SQL.
